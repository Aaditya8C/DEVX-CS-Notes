# Consistent Hashing

## Definition
Consistent hashing is a technique used to distribute data across multiple servers while minimizing data redistribution when servers are added or removed. It is widely used in distributed systems and caching infrastructures.

---

## Problem with Traditional Hashing

Traditional hashing distributes data using:

server = hash(key) % N

Where:
- key represents the data identifier
- N represents the number of servers

Issue:
When the number of servers changes, most keys are remapped to different servers, causing large-scale data migration and cache invalidation.

---

## Consistent Hashing Concept

Consistent hashing maps both servers and keys onto a circular hash space called a hash ring.

Characteristics:
- Servers are placed at positions on the ring based on their hash values
- Keys are also hashed and placed on the ring

---

## Data Placement Rule

A key is stored on the first server encountered while moving clockwise from the key's position on the hash ring.

This ensures deterministic placement and balanced distribution.

---

## Adding a Server

When a new server is added:
- Only the keys between the new server and its predecessor on the ring are reassigned
- The majority of keys remain unaffected

This significantly reduces data migration.

---

## Removing a Server

If a server fails or is removed:
- Its keys are reassigned to the next server clockwise on the ring

This ensures continued availability with minimal redistribution.

---

## Virtual Nodes

To improve load distribution, each physical server is represented by multiple virtual nodes on the hash ring.

Benefits:
- More uniform data distribution
- Reduced risk of load imbalance
- Improved scalability

---

## Advantages

- Minimal data movement when servers change
- Efficient horizontal scalability
- High fault tolerance
- Balanced data distribution

---

## Common Use Cases

Distributed databases:
- Cassandra
- DynamoDB

Distributed caching systems:
- Redis Cluster
- Memcached

Content delivery networks and large-scale storage systems.

---

## Key Takeaways

- Consistent hashing distributes keys across a circular hash space.
- Servers and keys are mapped to the same hash ring.
- Only a small subset of keys move when servers are added or removed.
- Virtual nodes improve load balancing and system scalability.
