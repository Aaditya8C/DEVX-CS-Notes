# Load Balancing

## Definition
Load balancing is the process of distributing incoming network traffic across multiple servers to ensure that no single server becomes overloaded. It improves system availability, scalability, reliability, and performance.

---

## Why Load Balancing is Needed

Without load balancing:

Client → Server

Problems:
- Server overload
- High latency
- Single point of failure

With load balancing:

Clients → Load Balancer → Multiple Servers

Benefits:
- Even traffic distribution
- Improved system reliability
- Better resource utilization

---

## Types of Load Balancers

### Hardware Load Balancer
Dedicated physical devices used for high-performance traffic distribution.

Advantages:
- Very high throughput
- Reliable

Disadvantages:
- Expensive
- Less flexible

---

### Software Load Balancer
Software-based solutions deployed on servers or cloud infrastructure.

Examples:
- Nginx
- HAProxy
- Envoy
- AWS Elastic Load Balancer

Advantages:
- Flexible
- Cost-effective
- Easy to scale

---

## Load Balancing Algorithms

### Round Robin
Requests are distributed sequentially among servers.

Characteristics:
- Simple implementation
- Does not consider server load

---

### Weighted Round Robin
Servers are assigned weights based on capacity. Servers with higher weights receive more requests.

---

### Least Connections
The request is sent to the server with the fewest active connections.

Advantage:
Better when request processing times vary.

---

### Least Response Time
Traffic is directed to the server with the lowest response time and fewest active connections.

---

## Layer 4 Load Balancing

Operates at the transport layer.

Routing decisions based on:
- IP address
- TCP/UDP ports

Advantages:
- Very fast
- Low overhead

---

## Layer 7 Load Balancing

Operates at the application layer.

Routing decisions based on:
- HTTP headers
- Cookies
- URL paths
- Request content

Advantages:
- Intelligent routing
- Content-based distribution

---

## Health Checks

Load balancers periodically verify server health.

If a server becomes unhealthy:
- It is removed from the pool
- Traffic is redirected to healthy servers

---

## Sticky Sessions (Session Affinity)

Ensures that requests from the same user are routed to the same server.

Techniques:
- Cookies
- Session IDs
- IP-based hashing

Drawback:
May cause uneven traffic distribution.

---

## Key Benefits

- Improved system availability
- Horizontal scalability
- Fault tolerance
- Reduced latency
- Efficient resource utilization
