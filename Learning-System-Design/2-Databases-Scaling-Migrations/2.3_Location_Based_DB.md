# Spatial Data & 2D Indexing Concepts

## Measurable Distance

### Definition
Distance between two points in space used to quantify closeness.

### Common Metrics

- Euclidean Distance:
  
  d = √((x2 - x1)^2 + (y2 - y1)^2)

- Manhattan Distance:

  d = |x2 - x1| + |y2 - y1|

---

### Example

Point A = (1,2)  
Point B = (4,6)

Euclidean Distance = √((4-1)^2 + (6-2)^2) = √(9 + 16) = √25 = 5

---

## Proximity

### Definition
Determines how "close" two points or objects are based on distance.

### Use Cases
- Nearby restaurants (maps)
- Ride matching (Uber/Ola)
- Location-based search

---

## Suitable Data Structures

For spatial data:

- KD Tree
- Quad Tree
- R Tree
- Grid Index
- Geohash / Space Filling Curves

---

## 2D Representation

Data stored as coordinate pairs:

(x, y)

Example:
- User location → (lat, lon)
- Game objects → (x, y)

---

## Bits for X, Y Axes

Coordinates can be represented in binary:

x = 5 → 101  
y = 3 → 011  

Used for:
- Space-filling curves
- Efficient indexing

---

## Searching in 2D

### Problem
Given a point or region, find nearby points.

---

### Naive Approach

- Scan all points

Time Complexity: O(N)

---

### Optimized Approach

Use spatial data structures:

- KD Tree → O(log N)
- Quad Tree → O(log N)

---

## Potential Drawback

- High-dimensional data reduces efficiency
- Boundary conditions complex
- Uneven data distribution causes imbalance
- Updating structures can be expensive

---

## Quad Trees

### Definition
A tree structure where each node divides space into 4 quadrants.

---

### Structure

Each node splits into:
- Top-left
- Top-right
- Bottom-left
- Bottom-right

---

### Example

2D space divided recursively until:
- Region contains few points
- Or reaches threshold

---

### Time Complexity

| Operation | Complexity |
|----------|-----------|
| Insert | O(log N) |
| Search | O(log N) |
| Range Query | O(log N + K) |

---

### Use Cases

- Image compression
- Spatial indexing
- Game collision detection

---

## Range Queries

### Definition
Retrieve all points within a given region.

Example:
- Find all users within 5 km radius

---

### Approaches

- Naive → O(N)
- Quad Tree / KD Tree → O(log N + K)

Where:
- K = number of results

---

## Fractals (2D → 1D Mapping)

### Definition
Technique to map multi-dimensional data into a single dimension while preserving locality.

Used in:
- Databases
- Caching
- Distributed systems

---

## Hilbert Curve

### Definition
A space-filling curve that maps 2D space into 1D while preserving locality.

Nearby points in 2D remain close in 1D.

---

## Why Hilbert Curve?

- Better locality than simple hashing
- Used in:
  - Databases
  - Geospatial indexing
  - Distributed storage

---

## Example

Points:

(0,0), (0,1), (1,1), (1,0)

Mapped order (Hilbert traversal):

(0,0) → (0,1) → (1,1) → (1,0)

---

## Comparison with Normal Ordering

Normal row-wise order:
- Breaks locality

Hilbert order:
- Preserves spatial proximity

---

## Use Cases

- Google Maps indexing
- Distributed databases
- Spatial search systems

---

## Key Takeaways

- Distance and proximity are core to spatial queries
- Quad Trees and KD Trees optimize 2D searches
- Range queries benefit from spatial indexing
- Space-filling curves map 2D → 1D efficiently
- Hilbert curve preserves locality better than naive methods
