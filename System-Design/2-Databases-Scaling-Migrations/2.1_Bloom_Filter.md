# Bloom Filter

## Definition
A Bloom Filter is a **space-efficient probabilistic data structure** used to test whether an element is **possibly present or definitely not present** in a set.

- False positives possible
- False negatives NOT possible

---

## Structure

- Bit array of size m
- k independent hash functions
- Initially all bits are 0

---

## Operations

### Insert(x)
- Compute k hash values: h1(x), h2(x), ..., hk(x)
- Set corresponding k positions in the bit array to 1

---

### Query(x)
- Check the same k positions
- If ANY bit = 0 → definitely not present
- If ALL bits = 1 → possibly present

---

## One-Hit Wonder Property

- Elements are not stored, only bits are updated
- Each insertion is a one-time operation on bits
- Bits once set cannot be unset (in standard Bloom Filter)

Implications:
- No deletion support
- False positive probability increases over time

---

## Probability Analysis

### Probability a bit is NOT set by one hash

P(bit not set) = (1 - 1/m)

---

### After k hash functions

P(bit not set after k hashes) = (1 - 1/m)^k

---

### After inserting n elements

Total hash operations = k × n

P(bit not set after n insertions):

P = (1 - 1/m)^(k * n)

---

### Approximation (for large m)

P ≈ e^(-kn/m)

---

### Probability that a bit IS set

P(bit set) = 1 - e^(-kn/m)

---

## False Positive Probability

A false positive occurs when all k bits are set for an element not present:

FP = (1 - e^(-kn/m))^k

---

## Optimal Number of Hash Functions

To minimize false positives:

k = (m/n) * ln(2)

---

## Key Insights

- Increasing m (bit array size) → reduces false positives
- Increasing k:
  - Too small → weak filtering
  - Too large → more collisions
- Increasing n → increases false positives

---

## Time Complexity

- Insert: O(k)
- Query: O(k)

---

## Advantages

- Very space efficient
- Fast operations
- No need to store actual elements

---

## Limitations

- False positives possible
- No deletion support (standard version)
- Cannot retrieve elements

---

## Variants

- Counting Bloom Filter (supports deletion)
- Scalable Bloom Filter
- Cuckoo Filter

---

## Use Cases

- Caching systems (avoid unnecessary DB queries)
- Databases (SSTable membership checks)
- Distributed systems
- Web crawling (URL deduplication)

---

## Key Takeaways

- Bloom Filter is a probabilistic membership structure
- Guarantees no false negatives
- False positives depend on m, n, k
- Widely used in large-scale systems
