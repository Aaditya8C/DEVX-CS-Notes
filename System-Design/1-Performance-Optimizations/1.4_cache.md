# Cache Memory - System Design Basics

## What is Cache?

Cache memory is a **small, high-speed memory layer** that stores copies of frequently accessed data or instructions so that future requests can be served faster.

It sits **between the CPU and main memory (RAM)** and significantly reduces the average data access time.

### Why Cache is Needed
- RAM access is slower compared to CPU speed
- Frequently used data can be served faster if stored in cache
- Reduces latency and improves overall system performance

---

## Memory Hierarchy
Registers → L1 Cache → L2 Cache → L3 Cache → Main Memory (RAM) → Secondary Storage (SSD / HDD)


Characteristics:

| Level | Speed | Size | Cost |
|------|------|------|------|
| Registers | Fastest | Smallest | Most Expensive |
| L1 Cache | Very Fast | Very Small | Expensive |
| L2 Cache | Fast | Small | Expensive |
| L3 Cache | Moderate | Larger | Cheaper |
| RAM | Slower | Large | Cheaper |
| Disk | Slowest | Very Large | Cheapest |

---

# Locality of Reference

Caching works based on the **principle of locality of reference**.

Programs tend to access the **same data or nearby data repeatedly**.

## Types of Locality

### Temporal Locality
If a memory location is accessed, it is likely to be accessed **again soon**.

Example:
for(int i=0;i<100;i++){
sum += arr[0];
}

`arr[0]` is accessed repeatedly.

---

### Spatial Locality

If a memory location is accessed, **neighboring locations** are likely to be accessed soon.

Example:
for(int i=0;i<100;i++){
sum += arr[i];
}

Access pattern:  
`arr[0] → arr[1] → arr[2]`

---

# Cache Performance

The effectiveness of cache is measured using the **Hit Ratio**.

### Hit
Data requested by CPU is **found in cache**.

### Miss
Data requested by CPU is **not found in cache** and must be fetched from RAM.

### Hit Ratio Formula
Hit Ratio = Hits / (Hits + Misses)


A **higher hit ratio means better performance**.

---

# Cache Line (Cache Block)

Cache does **not store individual bytes**, but instead stores **blocks of memory called cache lines**.

Typical cache line sizes:
- 32 bytes
- 64 bytes
- 128 bytes

Example:

If CPU requests address:
Address: 1000

Cache may load:
1000 - 1063 (64 bytes block)


This helps exploit **spatial locality**.

---

# Cache Mapping Techniques

These determine **where a memory block is placed in cache**.

## 1. Direct Mapped Cache

Each memory block maps to **exactly one cache line**.
Cache Line = Memory Address % Number of Cache Lines  <br></br>
Memory Block 1 → Cache Line 1 <br></br>
Memory Block 5 → Cache Line 1


Both compete for the same line.

---

## 2. Fully Associative Cache

Any memory block can be placed **in any cache line**.

### Advantages
- Most flexible
- Lowest conflict misses

### Disadvantages
- Expensive hardware
- Slow lookup

Cache must check **all lines simultaneously**.

---

## 3. Set Associative Cache

A **hybrid approach** combining direct and fully associative mapping.

Cache is divided into **sets**.

A memory block maps to a **specific set**, but can be placed in **any line inside that set**.

Example:
4-way set associative cache

Each set contains **4 cache lines**.

### Advantages
- Balanced performance
- Reduced conflict misses

Most modern CPUs use **set associative caches**.

---

# Cache Eviction Policies

When cache is **full**, a block must be removed to load new data.

This is called **cache eviction**.

## 1. LRU (Least Recently Used)

Evict the **least recently accessed** item.

Example:
Cache: A B C D
Access: A B C

Evict → D


Used in:
- Redis
- CPU caches

---

## 2. LFU (Least Frequently Used)

Evict the item with **least number of accesses**.


Evict **C**.

---

## 3. FIFO (First In First Out)

Evict the **oldest entry**.

Simple but less efficient.

---

# Write Policies

When CPU writes data, cache must decide **how to update memory**.

## Write Through

Write data to:
- Cache
- Main Memory

at the same time.

### Pros
- Data consistency

### Cons
- Slower writes

---

## Write Back (Write Behind)

Write only to cache first.

Memory is updated **later when block is evicted**.

### Pros
- Faster writes

### Cons
- More complex

Most systems use **write-back caching**.

---

# Multi-Level Cache

Modern CPUs use **multiple cache layers**.

## L1 Cache

- Closest to CPU
- Fastest
- Small (32KB – 128KB)

## L2 Cache

- Larger than L1
- Slightly slower

## L3 Cache

- Shared between CPU cores
- Much larger

Example:
CPU → L1 → L2 → L3 → RAM


---

# Types of Cache Misses

## 1. Cold Miss (Compulsory Miss)

Occurs when data is accessed **for the first time**.

Cache has never seen the data before.

---

## 2. Capacity Miss

Cache is **too small** to hold all needed data.

Older data must be evicted.

---

## 3. Conflict Miss

Occurs in **direct mapped caches** when multiple blocks compete for the same line.

---

# Example of Cache in System Design

Caching is widely used in **large-scale systems**:

Examples:
- Redis
- Memcached
- CDN caching
- Browser caching
- Database query caching

Example flow:
Client → Cache → Database

If data is in cache → **fast response**  
Else → fetch from DB → store in cache

---

# Key Takeaways

- Cache stores **frequently used data for faster access**
- Based on **temporal and spatial locality**
- Performance measured using **hit ratio**
- Uses **cache lines instead of bytes**
- Mapping methods:
  - Direct mapped
  - Fully associative
  - Set associative
- Eviction policies:
  - LRU
  - LFU
  - FIFO
- Write strategies:
  - Write-through
  - Write-back
